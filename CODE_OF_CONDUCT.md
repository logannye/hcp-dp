# Code Of Conduct

This project aims to be technically rigorous and practical. Contributions,
issues, and reviews should focus on evidence, reproducibility, and clear
tradeoffs.

Expected behavior:

- Be respectful and direct.
- Critique code, claims, tests, and documentation rather than people.
- Provide enough context for others to reproduce bugs or evaluate proposals.
- Accept corrections when evidence shows a claim or implementation is wrong.

Unacceptable behavior:

- Harassment, threats, or personal attacks.
- Dismissive or insulting comments toward contributors or users.
- Publishing private security details before a fix is available.
- Repeatedly derailing technical discussions after maintainers have redirected
  the thread.

Maintainers may edit, hide, or close issues and comments that violate these
expectations. Repeated or severe violations may result in blocked participation.
